## 1. 概述

### 1.1 什么是互动卡片

钉钉互动卡片是一种**即时交互、多人协同、数据驱动**的轻量卡片。它能够将原本复杂的应用解构成一个个轻量级的卡片，在钉钉的各个场域（如群聊、单聊、吊顶、工作台等）上运行。用户可以在卡片上完成互动协同，提高沟通效率，同时帮助业务更好地触达用户。

### 1.2 互动卡片三大特性

#### 1.2.1 数据驱动

- 互动卡片由**模板**和**数据**组成，数据决定了卡片的内容。
- 开发者只需搭建一次卡片模板，后续通过数据来驱动不同的卡片样式和内容。
- 卡片数据具备**动态性**：普通文本消息发送后内容无法变更，而互动卡片可以自由更新数据，让内容"动"起来。

#### 1.2.2 即时交互、多人协同

- 支持用户在卡片上进行轻量级交互，交互结果实时在卡片上同步显示。
- 用户无需打开额外页面，直接在钉钉聊天窗口即可完成所有操作。
- 同一张卡片的状态变更会实时同步给所有相关用户（如审批卡片：审批员操作后，发起人即可看到最新状态）。

#### 1.2.3 多场域运行

- 互动卡片不仅可以在聊天中运行，还可以运行在群吊顶、工作台、通讯录等场景。
- 同一个互动卡片模板可以同时创建消息卡片和吊顶卡片。

### 1.3 互动卡片类型

| 卡片类型 | 说明 | 适用场景 |
|---------|------|---------|
| **消息卡片** | 最常见的类型，以聊天消息形式出现在群聊或单聊中，拥有最丰富的组件和能力 | 群聊/单聊消息 |
| **标准卡片** | 通用卡片类型，会针对不同投放场景自适应能力与样式，组件较消息卡片少 | 跨场景同时投放（聊天、吊顶、协作、工作台等） |
| **吊顶卡片** | 以置顶消息形式出现在群聊中，尺寸固定 | 数据看板、公告通知 |
| **通讯录卡片** | 由组织管理员配置，插入到企业员工个人 Profile 中 | 员工数据展示或访问入口 |
| **工作台卡片** | 专用于工作台插件开发 | 工作台插件 |

### 1.4 核心名词解释

| 名词 | 说明 |
|------|------|
| **互动卡片** | 即时交互、多人协同、数据驱动的轻量卡片 |
| **搭建平台** | 卡片的低代码搭建平台，可进行模板搭建、管理、实例创建及投放 |
| **卡片模板** | 通过搭建平台创建的卡片模板样式，包含布局、组件、变量的集合 |
| **卡片实例化** | 通过模板和变量数据映射绑定生成卡片实例 |
| **卡片投放** | 将卡片实例投递到钉钉的开放场域（如群聊、单聊等） |
| **卡片更新** | 通过开放接口更新已投放卡片上的数据 |
| **事件回调** | 注册回调地址，当卡片发生互动行为时请求到业务服务 |
| **动态数据源** | 卡片渲染时实时发起请求到业务侧拉取数据，实现千人千面效果 |
| **场域** | 基于钉钉产品功能的协同场景，如群聊、单聊、个人 Profile 等 |
| **outTrackId** | 开发者自行生成的外部卡片实例唯一标识，一个 outTrackId 唯一标识一张卡片 |
| **openSpaceId** | 标识卡片投放的场域及其 ID，格式为 `dtv1.card//SpaceType.SpaceId` |

## 2. API 调用示例

本节提供常见的参数组合和各语言调用示例，帮助开发者快速上手。

### 2.1 场域类型：IM 机器人单聊

```http
POST /v1.0/card/instances/createAndDeliver HTTP/1.1
Host: api.dingtalk.com
x-acs-dingtalk-access-token: {你的access_token}
Content-Type: application/json

{
  "cardTemplateId": "{卡片模板id}",
  "outTrackId": "{唯一卡片实例id}",
  "cardData": {
    "cardParamMap": {
      "string": "字符串",
      "boolean": "true",
      "number": "1",
      "markdown": "# markdown",
      "obj": "{\"key\": \"value\"}",
      "arr": "[{\"key\": \"value\"}]"
    }
  },
  "openSpaceId": "dtv1.card//IM_ROBOT.{用户userId}",
  "imRobotOpenSpaceModel": {
    "supportForward": true
  },
  "imRobotOpenDeliverModel": {
    "spaceType": "IM_ROBOT"
  }
}
```

### 2.2 场域类型：IM 机器人群聊

```http
POST /v1.0/card/instances/createAndDeliver HTTP/1.1

Content-Type: application/json

{
  "cardTemplateId": "{卡片模板id}",
```

### 2.3 场域类型：吊顶

```http
POST /v1.0/card/instances/createAndDeliver HTTP/1.1
Host: api.dingtalk.com
x-acs-dingtalk-access-token: {你的access_token}
Content-Type: application/json

{
  "cardTemplateId": "{卡片模板id}",
  "outTrackId": "{唯一卡片实例id}",
  "cardData": {

      "string": "字符串",
      "boolean": "true",
      "number": "1"
    }
```

## 3. 卡片调试工具

### 3.1 调试入口

1. 登录[卡片平台](https://open.dingtalk.com)，点击「调试工具」。
2. 输入发出卡片的 **outTrackId** 进行查询。
3. 查询成功后，查询历史记录会刷新最新查询结果（保留最近 10 条记录）。
4. 点击绿色时间标签右边的操作按钮，可在「桌面端」或「移动端」打开卡片调试工具。

> **注意**：卡片调试工具目前只支持 IM 场域下的消息卡片、标准卡片、吊顶卡片、AI 卡片的调试，暂不支持通讯录卡片和工作台卡片。

### 3.2 调试功能

打开调试工具后，可以看到：

- **卡片基本信息**：查看卡片的模板 ID、实例 ID 等基础属性
- **卡片预览**：查看卡片的渲染效果
- **卡片数据**：查看下发到客户端上的卡片原始数据
- **数据校验**：查看当前卡片数据是否存在潜在问题（如变量值为空、变量类型不匹配等）
- **导出模板**按钮
- **清除模板缓存并刷新**按钮

> **重要**：调试工具只会获取客户端本地的卡片数据，无法获取在客户端上没有使用过的卡片数据。

### 3.3 清除模板缓存

开发过程中经常涉及模板更新，由于缓存机制可能导致端上使用的不是最新版本模板。操作步骤：

1. 修改卡片模板并保存。
2. 在卡片调试工具中点击「清除缓存并刷新」按钮。
3. 如果模板仍未更新，尝试重新发送一张卡片后切换会话，触发卡片重渲染拉取最新模板。

## 4. 创建卡片

### 4.1 接口说明

调用此接口可以从指定模板创建卡片实例，设置卡片的基本数据、动态数据源、卡片所在场域等。

- **请求方式**：`POST /v1.0/card/instances`
- **Host**：`api.dingtalk.com`
- **权限**：互动卡片实例写权限（企业内部应用、第三方企业应用均支持，第三方个人应用暂不支持）

### 4.2 请求头

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| x-acs-dingtalk-access-token | String | 是 | 访问凭证。企业内部应用调用获取企业内部应用的 accessToken 接口获取；第三方企业应用调用获取第三方应用授权企业的 accessToken 接口获取 |

### 4.3 核心 Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | String | 否 | 卡片创建者的 userId |
| cardTemplateId | String | **是** | 卡片内容模板 ID，通过开发者后台 > 卡片平台获取 |
| outTrackId | String | **是** | 外部卡片实例 ID，开发者自行生成。一个 outTrackId 唯一标识一张卡片。若需使用新的 cardTemplateId 或 cardData 创建新卡片，必须设置全新的 outTrackId |
| callbackType | String | 否 | 卡片回调类型：`STREAM`（Stream 模式）或 `HTTP`（HTTP 模式），注意参数均为大写 |
| callbackRouteKey | String | 否 | 卡片回调 HTTP 模式时的路由 Key，通过调用「注册卡片回调地址」接口获取 |
| cardData | Object | **是** | 卡片数据 |
| cardData.cardParamMap | Map<String, String> | 否 | 卡片模板内容替换参数。key：参数名（最长100B），value：参数值（最长1KB）。所有属性字段只支持 String 类型，非 String 类型请参考「API 卡片数据的填写说明」 |
| privateData | Map<String, Object> | 否 | 用户的私有数据。key：用户 userId，value：该用户的私有数据对象（包含 cardParamMap） |
| userIdType | Integer | 否 | 用户 ID 类型：1（默认，userId 模式）或 2（unionId 模式） |

### 4.4 动态数据源配置（openDynamicDataConfig）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| dynamicDataSourceConfigs | Array | 否 | 动态数据源配置列表 |
| dynamicDataSourceId | String | 否 | 数据源唯一 ID（使用动态数据源时必填） |
| constParams | Map<String, String> | 否 | 回调数据源时回传的固定参数 |
| pullConfig.pullStrategy | String | 否 | 拉取策略：`NONE`（不拉取）、`INTERVAL`（间隔拉取）、`ONCE`（只拉取一次）。使用动态数据源时必填 |
| pullConfig.interval | Integer | 否 | 拉取间隔时间（仅 INTERVAL 模式生效，最小 3s） |
| pullConfig.timeUnit | String | 否 | 间隔时间单位：`SECONDS`、`MINUTES`、`HOURS`、`DAYS` |

### 4.5 场域信息模型

创建卡片时可以指定场域信息（以下均为可选）：

#### IM 单聊酷应用场域（imSingleOpenSpaceModel）

| 参数名 | 类型 | 说明 |
|--------|------|------|
| supportForward | Boolean | 是否支持转发，默认 false |
| lastMessageI18n | Map<String, String> | 国际化 LastMessage，支持 ZH_CN / ZH_TW / EN_US / JA_JP / VI_VN / TH_TH |
| searchSupport.searchIcon | String | 搜索图标 |
| searchSupport.searchTypeName | String | 卡片类型名 |
| searchSupport.searchDesc | String | 搜索描述（最大200字符） |
| notification.alertContent | String | 通知内容，不填使用默认文案 |
| notification.notificationOff | Boolean | 是否关闭推送通知，默认 false |

#### IM 群聊场域（imGroupOpenSpaceModel）

与单聊酷应用场域参数一致，其中 **supportForward 为必填**。

#### IM 机器人单聊场域（imRobotOpenSpaceModel）

与单聊酷应用场域参数一致，其中 **supportForward 为必填**。

#### 吊顶场域（topOpenSpaceModel）

| 参数名 | 类型 | 说明 |
|--------|------|------|
| spaceType | String | 吊顶场域属性，固定为 `ONE_BOX`（必填） |

#### @人功能（cardAtUserIds）

- 类型：Array of String
- 设置被 @人的 userId 列表
- 需在卡片 markdown 内容中添加：`<a atId=example_user_id>用户昵称</a>`

### 4.6 返回参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 创建是否成功 |
| result | String | outTrackId，外部卡片实例 ID |

### 4.7 请求示例

```http
POST /v1.0/card/instances HTTP/1.1
Host: api.dingtalk.com
x-acs-dingtalk-access-token: token-xxxx
Content-Type: application/json

{
  "userId": "example_user_id",
  "cardTemplateId": "b4fdsu2119f-9945-4e13-9989-747da19e3bc7",
  "outTrackId": "example_out_track_id",
  "callbackType": "STREAM",
  "callbackRouteKey": "example_route_key",
  "cardData": {
    "cardParamMap": {
      "key": "example_public_value"
    }
  },
  "privateData": {
    "manager1234": {
      "cardParamMap": {
        "key": "example_private_value"
      }
    }
  }
}
```

### 4.8 常见错误码

| HttpCode | 错误码 | 说明 |
|----------|--------|------|
| 400 | param.cardTemplateIdEmpty | 卡片模板 ID 为空 |
| 400 | param.outTrackIdEmpty | outTrackId 为空 |
| 400 | param.cardAlreadyExist | 卡片已存在（outTrackId 重复） |
| 400 | param.templateNotExist | 模板不存在 |
| 400 | param.templateUnpublished | 模板未发布 |
| 400 | param.contentUnsafe | 卡片数据未通过安全审查 |
| 400 | param.openSpaceModelInvalid | 错误的场域属性模型 |
| 400 | param.dynamicSourceIdEmpty | 动态数据源配置 ID 为空 |
| 500 | system.busy | 系统繁忙 |

## 5. 投放卡片

### 5.1 接口说明

调用此接口将已创建的卡片实例投放至多个指定场域。

- **请求方式**：`POST /v1.0/card/instances/deliver`
- **Host**：`api.dingtalk.com`
- **权限**：互动卡片实例写权限
- **支持场域**：IM 群聊、IM 单聊酷应用、IM 机器人单聊、吊顶、协作、文档

### 5.2 场域类型与 SpaceId 定义

| 场域类型 | SpaceType | SpaceId | SpaceId 含义 |
|----------|-----------|---------|-------------|
| IM 群聊 | IM_GROUP | openConversationId | 会话 ID |
| IM 单聊酷应用 | IM_SINGLE | openConversationId | 会话 ID |
| IM 机器人单聊 | IM_ROBOT | userId / unionId | 员工 ID |
| 吊顶 | ONE_BOX | openConversationId | 会话 ID |

**openSpaceId 格式**：`dtv1.card//SpaceType.SpaceId`

示例：IM 群聊的 openSpaceId 为 `dtv1.card//IM_GROUP.cidg2bR***JzmpFY=`，其中 `dtv1.card//` 是固定前缀，`IM_GROUP` 是群聊标识，后面是群会话的 openConversationId。

**多场域投放格式**：`dtv1.card//spaceType1.spaceId1;spaceType2.spaceId2`

> **注意**：若需投放的卡片实例不支持该场域，需先调用 appendSpace 接口（新增场域信息接口）增加该场域。将卡片投放至文档不需要此步骤。

### 5.3 核心 Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| outTrackId | String | **是** | 外部卡片实例 ID |
| openSpaceId | String | **是** | 场域及场域 ID |

### 5.4 各场域投放参数

#### 单聊场域（imSingleOpenDeliverModel）

| 参数名 | 类型 | 说明 |
|--------|------|------|
| atUserIds | Map<String, String> | @人。key: userId, value: 用户名。key 和 value 都为 "@ALL" 则 @所有人 |
| extension | Map<String, String> | 扩展字段 |

#### 机器人单聊（imRobotOpenDeliverModel）

| 参数名 | 类型 | 说明 |
|--------|------|------|
| spaceType | String | 若未设置其他投放属性，需设为 `IM_ROBOT` |
| robotCode | String | 机器人编码 |
| extension | Map<String, String> | 扩展字段 |

#### 群聊（imGroupOpenDeliverModel）

| 参数名 | 类型 | 说明 |
|--------|------|------|
| robotCode | String | **必填**。场景群机器人用群机器人 robotCode；企业内部机器人用 AppKey；第三方企业机器人用 robotCode |
| atUserIds | Map<String, String> | @人 |
| recipients | Array of String | 指定接收者的 userId |
| extension | Map<String, String> | 扩展字段 |

#### 吊顶（topOpenDeliverModel）

| 参数名 | 类型 | 说明 |
|--------|------|------|
| expiredTimeMillis | Long | **必填**。过期时间戳 |
| userIds | Array of String | 可查看该吊顶卡片的 userId 列表 |
| platforms | Array of String | 可查看的设备：android / ios / win / mac。为空则所有设备可见 |

### 5.5 返回参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 调用结果 |
| result | Array | 投放结果列表 |
| result[].spaceType | String | 场域类型 |
| result[].spaceId | String | 场域 ID |
| result[].success | Boolean | 投放是否成功 |
| result[].carrierId | String | 投放结果 ID（IM 场域返回 processQueryKey，可用于查看消息已读列表） |
| result[].errorMsg | String | 错误信息 |

### 5.6 常见错误码

| HttpCode | 错误码 | 说明 |
|----------|--------|------|
| 400 | param.outTrackIdEmpty | outTrackId 为空 |
| 400 | param.openSpaceIdEmpty | openSpaceId 为空 |
| 400 | param.openSpaceIdInvalid | openSpaceId 格式不规范 |
| 400 | param.cardNotExist | 卡片不存在 |
| 400 | param.openDeliverModelEmpty | 场域投放模型为空 |
| 500 | system.busy | 系统繁忙 |

## 6. 更新卡片

### 6.1 接口说明

调用此接口主动更新已投放卡片的数据。

- **请求方式**：`PUT /v1.0/card/instances`
- **Host**：`api.dingtalk.com`
- **权限**：互动卡片实例写权限

### 6.2 核心 Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| outTrackId | String | **是** | 外部卡片实例 ID |
| cardData | Object | 否 | 卡片公共数据 |
| cardData.cardParamMap | Map<String, String> | 否 | 模板内容替换参数（key 最长100B，value 最长1KB） |
| privateData | Map<String, Object> | 否 | 用户私有数据（key: userId, value: 用户私有数据对象） |
| cardUpdateOptions | Object | 否 | 卡片更新选项 |
| userIdType | Integer | 否 | 用户 ID 类型：1（userId）或 2（unionId） |

### 6.3 更新选项（cardUpdateOptions）

| 参数名 | 类型 | 默认值 | 说明 |

| updatePrivateDataByKey | Boolean | false（覆盖更新） | `true`：按 key 增量更新 privateData；`false`：覆盖更新 privateData |

> **重要区别**：
> - **覆盖更新**（默认）：每次更新会用新数据完全替换旧数据，未传的字段会被清空。
> - **按 key 更新**：仅更新传入的字段，其余字段保持不变。推荐在大多数场景下使用按 key 更新。

## 9. AI 卡片流式更新

### 9.1 接口说明

AI 卡片流式更新接口旨在为 AIGC 产生的内容提供持续更新的能力。通过此接口持续更新的内容，在客户端会呈现**打字机效果**。

- **请求方式**：`PUT /v1.0/card/streaming`
- **Host**：`api.dingtalk.com`
- **权限**：AI 卡片流式更新权限（企业内部应用、第三方企业应用支持）

### 9.2 Body 参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| outTrackId | String | **是** | 外部卡片实例 ID，需与创建卡片时的 outTrackId 一致 |
| guid | String | **是** | 请求调用的唯一标志，用于幂等判断 |
| key | String | **是** | 需要进行流式更新的变量名 |
| content | String | **是** | 本次更新的流式内容。单次不超过 1KB，总大小建议不超过 3KB |
| isFull | Boolean | 否 | 是否全量更新。`true`：覆盖更新；`false`（默认）：增量追加。**若变量绑定的是 markdown，必须设为 true** |
| isFinalize | Boolean | 否 | 是否为最后一帧。`true`：AI 卡片从「输入中」切换为「完成」状态 |
| isError | Boolean | 否 | 是否出错。`true`：AI 卡片从「输入中」切换为「出错」状态 |

### 9.3 关键注意事项

1. **Markdown 内容**：由于服务端需进行格式转换，markdown 内容必须是全量内容且语法完整，`isFull` 必须设为 `true`。
2. **内容大小**：单次 content 不超过 1KB，总大小建议不超过 3KB。
3. **状态控制**：通过 `isFinalize` 和 `isError` 控制 AI 卡片的状态转换。
4. **幂等性**：`guid` 用于保证同一请求不会被重复处理。

### 9.4 请求示例

```http
PUT /v1.0/card/streaming HTTP/1.1
Host: api.dingtalk.com
x-acs-dingtalk-access-token: your-access-token
Content-Type: application/json

{
  "outTrackId": "your-out-track-id",
  "guid": "0F714542-0AFC-2B0E-CF14-E2D39F5BFFE8",
  "key": "your-ai-param",
  "content": "这是一段AI生成的内容",
  "isFull": false,
  "isFinalize": false,
  "isError": false
}
```

### 9.5 典型流式更新流程

1. 创建并投放一张 AI 卡片（使用 AI 卡片模板）。
2. 循环调用流式更新接口，每次传入一小段内容（如逐字/逐句），每次调用需更换 `guid`。
3. 最后一次调用时设置 `isFinalize: true`，结束流式更新。
4. 若发生错误，设置 `isError: true`，卡片展示错误状态。

### 9.6 常见错误码

| HttpCode | 错误码 | 说明 |
|----------|--------|------|
| 400 | param.stream.keyEmpty | 更新 key 为空 |
| 400 | param.stream.contentEmpty | 更新 content 为空 |
| 400 | param.stream.guidEmpty | guid 为空 |
| 400 | param.stream.outTrackId | 卡片不存在 |
| 400 | param.stream.isFull | isFull 字段为空 |
| 400 | param.stream.content | content 过大 |

### 10.1 接口说明

| apiSecret | String | 否 | 加密密钥，用于校验回调来源 |

### 10.3 返回参数

### 10.5 回调机制说明

注册回调地址后，在创建卡片时通过以下方式关联：

- **HTTP 模式**：创建卡片时设置 `callbackType: "HTTP"`，并设置 `callbackRouteKey` 为注册时使用的 key。

当用户在卡片上触发交互事件（如点击按钮），钉钉服务端会向注册的 callbackUrl 发送 HTTP 请求，开发者的业务服务收到请求后进行处理并返回结果。

## 13. API 卡片数据的填写说明

创建卡片、创建并投放卡片、更新卡片三个接口都使用 `Map<String, String>` 来传递卡片数据，Key 为参数名、Value 为参数值。由于 Value 类型固定为 String，非 String 类型需要特殊处理。

### 13.1 String 类型参数

直接填写参数名和参数值：

```json
{
  "outTrackId": "testId",
  "cardData": {
    "cardParamMap": {
      "strParam1": "str1",
      "strParam2": "str2"
    }
  },
  "privateData": {
    "myUserId": {
      "cardParamMap": {
        "privateStrParam3": "str3",
        "privateStrParam4": "str4"
      }
    }
  }
}
```

### 13.2 基本类型（Number、Boolean）

将数值和布尔值转为字符串：

```json
{
  "cardData": {
    "cardParamMap": {
      "intParam": "1",
      "floatParam": "1.23",
      "trueParam": "true",
      "falseParam": "false"
    }
  }
}
```

### 13.3 对象类型（Object）

将对象转为 JSON 字符串：

```json
{
  "cardData": {
    "cardParamMap": {
      "objParam": "{\"strParam\":\"stringValue\",\"boolParam\":true,\"floatParam\":1.2,\"intParam\":123,\"innerObjParam\":{\"innerStrParam\":\"stringValue\",\"innerIntParam\":123}}"
    }
  }
}
```

上述 `objParam` 解析后的 JSON 对象：

```json
{
  "strParam": "stringValue",
  "boolParam": true,
  "floatParam": 1.2,
  "intParam": 123,
  "innerObjParam": {
    "innerStrParam": "stringValue",
    "innerIntParam": 123,
    "innerBoolParam": true
  }
}
```

### 13.4 数组类型（Array）

将数组转为 JSON 字符串：

```json
{
  "cardData": {
    "cardParamMap": {
      "intArrParam": "[1,2,3]",
      "boolArrParam": "[true,false]",
      "objArrParam": "[{\"param1\":\"val1\"},{\"param2\":\"val2\"}]",
      "floatArrParam": "[1.1,2.2,3.3]",
      "stringArrParam": "[\"str1\",\"str2\"]"
    }
  }
}
```

> **核心规则总结**：
> - String 类型：直接填写字符串值
> - Number / Boolean 类型：转为字符串（如 `"1"`、`"true"`）
> - Object / Array 类型：先序列化为 JSON 字符串再填入
> - 务必确保值类型与卡片搭建器中配置的变量类型匹配，否则可能导致属性不生效或移动端无法显示

## 14. 常见问题

### 14.1 卡片显示不符合预期

使用卡片调试工具（参考第 3 节）查看下发的卡片数据是否正确。

### 14.2 非 String 类型参数如何处理

- **新版接口**（创建卡片、创建并投放卡片、更新卡片）：按照第 13 节「API 卡片数据的填写说明」处理。
- **旧版接口**（如机器人发送钉钉互动卡片高级版）：使用 `sys_full_json_obj` 字段，将所有非 String 参数构建成 JSONObject，再转为 JSONString 填入该字段：

```json
{
  "cardData": {
    "cardParamMap": {
      "sys_full_json_obj": "{\"intParam\":1,\"floatParam\":1.2345,\"boolParam\":true,\"strArr\":[\"str1\",\"str2\"],\"objParam\":{\"objField1\":3,\"objField2\":3.14}}"
    }
  }
}
```

### 14.3 创建/更新卡片后参数值未正常显示

- 确保非 String 类型参数按照规范正确设置。
- 核对卡片模板搭建器中配置的变量类型与传入的属性值类型是否匹配。

### 14.4 卡片模板或数据更新后发送的卡片没有更新

检查是否更换了 `outTrackId`。如果使用了新的 `cardTemplateId` 或 `cardData`，必须生成全新的 `outTrackId`，否则更改不会生效。

### 14.5 cardData 中包含 Markdown 如何入参

示例：`"cardData": "{\"field_1\":\"**钉钉平台<font color=common_blue1_color>已发起</font>**\"}"`

### 14.6 修改模板后发送的卡片没有更新

卡片模板有缓存，可以重启钉钉后重试，或者重发一张新的卡片后重进会话试试。

### 14.7 卡片回传请求事件返回的私有数据没生效

回传请求返回的私有数据要设置在 `userPrivateData` 字段下面，且不需要以 userId 为 key：

```json
{
  "userPrivateData": {
    "cardParamMap": {
      "key": "value"
    }
  }
}
```

### 14.8 定义了 privateData 但卡片没有更新

需进入卡片搭建平台 > 数据源 > 编辑，选择对应的字段**开启私有变量**即可正常更新。

### 14.9 如何给不支持点击事件的元素绑定点击事件

使用「单个容器」组件来包裹目标元素（如文本组件），然后在「单个容器」组件上绑定点击事件。

### 14.10 卡片发布与不发布的区别

- **未发布**的卡片可多次修改，但每次修改会直接影响线上所有卡片，容易导致线上故障。
- **已发布**的卡片受到保护，不会因意外操作导致故障。
- **建议**：业务卡片正式上线后要及时发布模板。

### 14.11 互动卡片高级版与普通版区别

| 能力 | 普通版 | 高级版 |
|------|--------|--------|
| 搭建平台提取操作代码 | 支持 | 不支持 |
| 搭建平台导出 | JSON 可一键复制导出 | 可一键导出 |
| 互动卡片回调 | 不支持 | **支持** |
| 私有数据 | 不支持 | **支持** |
| 机器人发送/更新互动卡片 | 支持 | 支持 |
| 组件/区块组件库 | 支持 | 支持 |
| 一键推送测试卡片 | 支持 | 不支持 |
| 表情 | 支持 | 支持 |

### 14.12 userIdType 字段填写说明

`userIdType` 控制整个请求中用户 ID 的类型：

| 值 | 模式 | 获取方式 |
|----|------|---------|
| 1（默认） | userId 模式 | 调用获取部门用户 userId 列表接口 |
| 2 | unionId 模式 | 调用通过免登码获取用户信息接口 |

**选取规则**：
- 企业内部创建且仅供内部使用 → 使用 **userId 模式**
- 企业内部创建但也提供给外部人员使用（如服务群） → 使用 **unionId 模式**
- ISV 创建的卡片 → 使用 **unionId 模式**

### 14.13 如何关闭吊顶卡片

1. **搭建关闭按钮 UI**：进入卡片搭建平台，使用吊顶卡片模板（自带关闭按钮）或使用图片组件自行搭建。
2. **设置回传请求**：选择"X"图片组件所在的布局 > 点击"X" > 点击事件类型选择"回传请求" > 设置回传参数。
3. **调用关闭接口**：通过服务端 API（第 8 节）实现关闭。

> **注意**：需先调用「注册卡片回调地址」接口设置回传请求。群成员关闭卡片后所有人不可见，建议设置关闭按钮仅对管理员可见。

### 14.14 卡片组件动态可见

可以实现。设置步骤：数据源 > 编辑变量 > 按钮的点击事件类型选择「回传请求」 > 设置回传参数 > 设置「是否显示」选择「条件计算」 > 创建新条件。当变量（如 isShow）为 true 时显示，为 false 时隐藏。

### 14.15 添加多个按钮提示不能添加

横排或竖排按钮的添加在按钮设置最下方「添加新按钮」处操作。竖排按钮不支持显示隐藏配置，如需该功能请使用普通组件实现。

### 14.16 卡片多媒体资源如何使用

多媒体资源支持 HTTP 协议，可以直接使用业务可用的 HTTP 资源链接。

### 14.17 Windows 和安卓卡片显示 [Object Object]

卡片测试期间因数据和模板更新，可能出现不兼容老版本模板的情况，重启钉钉即可重新拉取模板。

## 附录 A：完整开发流程总览

以下是使用钉钉互动卡片的端到端开发流程：

### 步骤一：准备工作

1. 登录[钉钉开发者后台](https://open.dingtalk.com)，创建企业内部应用或第三方企业应用。
2. 获取应用的 **AppKey** 和 **AppSecret**。
3. 调用获取 accessToken 接口，获取 `x-acs-dingtalk-access-token`。
4. 在应用权限管理中，申请「互动卡片实例写权限」。

### 步骤二：搭建卡片模板

1. 进入开发者后台 > [卡片平台](https://open.dingtalk.com)。
2. 创建卡片模板，选择模板类型（消息卡片 / 标准卡片 / 吊顶卡片等）。
3. 使用可视化搭建器设计卡片布局、添加组件、配置变量。
4. 在数据源中定义变量（String / Number / Boolean / Object / Array 类型）。
5. 若需要私有数据，在数据源中开启对应变量的「私有变量」开关。
6. 若需要交互功能，为按钮等组件绑定点击事件（回传请求 / 跳转链接）。
7. 保存并发布模板，记录 **卡片模板 ID（cardTemplateId）**。

### 步骤三：注册回调地址（如需交互）

若卡片需要交互功能（如按钮点击回调），需先注册回调地址：

- **HTTP 模式**：调用注册卡片回调地址接口（第 10 节），设置 callbackRouteKey 和 callbackUrl。
- **STREAM 模式**：使用 Stream 通道接收回调，无需注册 URL。

### 步骤四：创建并投放卡片

推荐使用「创建并投放卡片」接口（第 7 节）一步到位：

1. 生成唯一的 `outTrackId`（如 UUID）。
2. 构造 `cardData`，按照第 13 节的规范填写各类型参数。
3. 构造 `openSpaceId`，格式为 `dtv1.card//SpaceType.SpaceId`。
4. 根据目标场域设置对应的 SpaceModel 和 DeliverModel。
5. 调用接口发送卡片。

或者分两步操作：先调用「创建卡片」接口（第 4 节），再调用「投放卡片」接口（第 5 节）。

### 步骤五：更新卡片（可选）

需要更新已发送的卡片时，调用「更新卡片」接口（第 6 节），通过 outTrackId 定位卡片，传入新的 cardData。建议使用 `updateCardDataByKey: true` 进行按 key 增量更新。

### 步骤六：处理回调事件（如需交互）

用户在卡片上触发交互后，钉钉会向注册的回调地址发送请求。开发者的业务服务需要：

1. 接收并解析回调请求。
2. 根据业务逻辑处理交互事件。
3. 返回处理结果（可选择返回更新后的卡片数据）。

### 步骤七：调试

使用卡片调试工具（第 3 节）排查问题：

1. 在卡片平台打开调试工具。
2. 输入 outTrackId 查询卡片。
3. 查看卡片数据、数据校验结果。
4. 如有模板缓存问题，点击「清除缓存并刷新」。
