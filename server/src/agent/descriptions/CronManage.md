Create, update, delete or immediately run a scheduled cron job owned by the current user. Use this whenever the user asks for a recurring task, a scheduled reminder, or a one-time delayed action. action=create requires name + schedule + payload. Schedule kinds: cron (5-field expr, e.g. "0 9 * * *" with tz "Asia/Shanghai"), every (fixed interval in ms), at (one-shot epoch ms). Payload kinds: agentTurn (the platform starts a fresh agent session as the job owner and sends the message — use this for reminders and automated work) or systemEvent (plain notification text). Add notify {"enabled":true,"channel":"web"} so the user sees the result push; only use the dingtalk channel when the user explicitly provides the target. action=update patches only the provided fields of an existing job; action=delete removes it; action=run triggers it once immediately. Only jobs owned by the current user can be read or modified.
