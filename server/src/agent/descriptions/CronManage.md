创建、更新、删除或立即运行当前用户拥有的定时任务（cron job）。当用户要求周期性任务、定时提醒或一次性延时动作时使用。action=create 需要 name + schedule + payload。schedule 类型：cron（5 段表达式，如 "0 9 * * *"，配 tz "Asia/Shanghai"）、every（固定间隔，单位 ms）、at（一次性，epoch ms）。payload 类型：agentTurn（平台以任务所有者身份启动全新 agent 会话并发送该消息——提醒和自动化工作用这个）或 systemEvent（纯通知文本）。加上 notify {"enabled":true,"channel":"web"} 让用户收到结果推送；仅当用户明确提供目标时才使用 dingtalk 渠道。action=update 只修改传入的字段；action=delete 删除任务；action=run 立即触发一次。只能读取和修改当前用户自己的任务。
