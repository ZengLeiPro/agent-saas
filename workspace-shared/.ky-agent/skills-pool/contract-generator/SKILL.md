# 销售合同生成器

为开沿科技的定制化/钉钉业务客户生成 Word 格式的销售合同（钉钉合作服务协议）。

## 第一步：信息采集（必须执行）

收到合同需求后，**先分析用户已提供的信息，识别缺失项，使用 AskUserQuestion 工具一次性提问**。不要边做边问，也不要在信息不全时就开始生成。

### 必须确认的信息

| 信息 | 判断逻辑 |
|------|---------|
| 甲方公司全称 | 用户未提及 -> 必问 |
| 服务项目明细 | 用户未列明具体项目、内容和描述 -> 必问 |
| 服务期限 | 用户未说明起止日期 -> 必问 |
| 合同总价 | 用户未提供金额 -> 必问 |
| 项目负责人 | 用户未指定 -> 必问（参考下方人员速查表） |

### 可以智能默认的信息

| 信息 | 默认值 | 何时需要问 |
|------|--------|----------|
| 乙方公司名 | 福建开沿科技有限公司 | 几乎不需要问 |
| 付款方式 | 合同签订之日起五个工作日内一次性支付 | 用户有特殊付款安排时才问 |
| 收款银行信息 | 招商银行泉州泉秀支行（固定） | 不需要问 |
| 合同标题 | 钉钉合作服务协议 | 用户提到非钉钉项目时问 |

### 提问示例

当用户说"帮我给XX公司出个合同"但信息不足时：

> 合同需要以下信息，请补充：
> 1. **服务项目**：具体有哪些服务？每项的服务内容和业务描述是什么？
> 2. **服务期限**：从什么时候到什么时候？（例如：2026年4月1日至2027年3月31日）
> 3. **合同总价**：总金额是多少？
> 4. **项目负责人**：哪位同事负责？（彭一宁 / 黄思霖 / 陈育新 / 许锐宏）

当信息比较完整时（如"帮我出个合同，甲方泉州XX公司，钉钉专业版 9800 一年，彭一宁负责，4月1号到明年3月31号"），直接组装 JSON 生成。

## 第二步：生成合同

信息齐全后：
1. 组装 JSON 数据文件，写入 `assets/contracts/` 目录
2. 调用生成脚本，输出 .docx + .json 两个文件
3. 将 .docx 文件交付给用户

## 调用方式

```bash
node SKILL_DIR/scripts/generate_contract.js <input.json> [output_dir]
```

- `SKILL_DIR` = 本 skill 的绝对路径（即本文件所在目录）
- `input.json` = 合同数据 JSON 文件（结构见下方 Schema）
- `output_dir` = 输出目录，默认 `assets/contracts/`（目录不存在时自动创建）

输出文件命名：`{甲方名}_合同.docx` / `.json`

## JSON Schema

```json
{
  "partyA": "甲方公司全称（必填）",
  "title": "合同标题（选填，默认：钉钉合作服务协议）",
  "services": [
    {
      "name": "服务项目名称（必填）",
      "content": "服务内容（必填，支持多行用 \\n 分隔）",
      "description": "业务描述及实现方式（必填）"
    }
  ],
  "period": {
    "start": "服务起始日期（必填，格式：YYYY年MM月DD日）",
    "end": "服务结束日期（必填，格式：YYYY年MM月DD日）"
  },
  "totalAmount": 9800,
  "projectManager": {
    "name": "项目负责人姓名（必填）",
    "phone": "联系电话（必填）"
  },
  "paymentTerms": "付款方式描述（选填，默认：合同签订之日起五个工作日内，甲方向乙方一次性支付合同总款项）"
}
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `partyA` | YES | 甲方公司全称 |
| `title` | - | 合同标题，默认"钉钉合作服务协议" |
| `services` | YES | 服务项目数组，至少 1 个 |
| `services[].name` | YES | 服务项目名称 |
| `services[].content` | YES | 服务内容，支持 `\n` 换行 |
| `services[].description` | YES | 业务描述及实现方式 |
| `period.start` | YES | 格式 `YYYY年MM月DD日` |
| `period.end` | YES | 格式 `YYYY年MM月DD日` |
| `totalAmount` | YES | 合同总价，数字类型（脚本自动转大写） |
| `projectManager.name` | YES | 项目负责人姓名 |
| `projectManager.phone` | YES | 项目负责人电话 |
| `paymentTerms` | - | 自定义付款方式描述 |

## 修改已有合同

1. 找到对应的 `.json` 文件（`assets/contracts/` 目录下按甲方名搜索）
2. 修改 JSON 内容
3. 重新运行生成脚本，会覆盖同名文件

## 注意事项

- `totalAmount` 必须是数字，不要用字符串（正确：`9800`，错误：`"9,800"`）
- 脚本会自动将数字金额转换为中文大写（如 9800 -> 玖仟捌佰元整）
- 合同条款（第四至六条）为固定标准条款，不需要在 JSON 中指定
- 乙方银行账户信息已硬编码在脚本中（招商银行泉州泉秀支行 595902938010801）
- 生成依赖 Node.js 和 docx 包（`npm install -g docx`，已安装）
