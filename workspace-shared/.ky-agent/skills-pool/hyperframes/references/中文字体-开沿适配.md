# 中文字体 — 开沿适配指南（HyperFrames）

> HyperFrames 内置字体里没有中文。做中文视频必须自带中文字体文件，并在 composition 中用本地 `@font-face` 引入。Google Fonts 外链只可作为临时探索，不可作为 ACS 交付方案。

## 1. 为什么要专门处理中文

1. **内置字体里没有中文**：producer 预置内嵌字体里 CJK 只有日文 `noto-sans-jp` / `noto-sans-japanese`，没有 Noto Sans SC/TC、思源黑体或其他中文字体。
2. **ACS 交付必须自包含**：最终 HTML/视频渲染不应依赖 Google Fonts、CDN、远程字体或运行时外网可用性。外链字体会让同一 composition 在不同网络状态下出现不同结果。
3. **不处理的后果**：既不自带中文字体、也没有可用系统中文字体时，中文会回落到 generic 字体，表现为豆腐块、缺字、字重错误或字形不一致。

## 2. 推荐路径：本地 woff2

1. 准备开源可商用中文字体，如思源黑体 / Noto Sans SC。优先用 `.woff2`；只有 `.otf`/`.ttf` 时先转换或子集化。
2. 放进项目 `fonts/` 目录，与 `index.html` 同级。
3. 在 HTML/CSS 中写本地 `@font-face`：

```css
@font-face {
  font-family: "Noto Sans SC";
  src: url("fonts/NotoSansSC-Regular.woff2") format("woff2");
  font-weight: 400;
  font-display: block;
}
@font-face {
  font-family: "Noto Sans SC";
  src: url("fonts/NotoSansSC-Bold.woff2") format("woff2");
  font-weight: 700;
  font-display: block;
}

body {
  font-family: "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif;
}
```

4. 对所有中文文本、字幕、数据标签使用同一个中文字体族；数字列加 `font-variant-numeric: tabular-nums`。

## 3. 子集化控体积

整字库通常有数 MB，会拖慢渲染和文件加载。文案确定后，按实际用字做子集：

```bash
python3 -m pip install fonttools brotli
pyftsubset NotoSansSC-Regular.otf \
  --text-file=文案.txt \
  --flavor=woff2 \
  --output-file=fonts/NotoSansSC-Regular.subset.woff2
```

安装限制：

- 只允许装进 workspace `.venv`，不要用 `sudo pip`、`pip install --user`、`--break-system-packages`，也不要新建 venv。
- 如果 `.venv` 不可用或缺少构建依赖，停止并报告 runtime image 缺口。

批量渲染注意：

- 使用 `--variables` 生成多变体时，子集必须覆盖所有变体可能出现的字。
- 不确定全集时，用完整字库或足够大的常用字集，不要为了几十 KB 冒缺字风险。

## 4. Google Fonts 的位置

Google Fonts 可用于早期字体探索或临时预览，但不能作为最终交付依赖：

- 不要在最终 HTML 里保留 `fonts.googleapis.com` / `fonts.gstatic.com`。
- 不要把 `google_fonts_import` 当成可忽略 warning；在 ACS 交付语境里它就是要修的外部依赖信号。
- 如果设计来自 Google Fonts，下载对应开源字体文件，放入项目 `fonts/`，再改成本地 `@font-face`。

## 5. 交付前实测清单

- [ ] `npx hyperframes inspect` 或截图检查每一类中文画面，无豆腐块、无缺字。
- [ ] 简体 / 繁体（若有）字形正确。
- [ ] 常规、粗体等字重都有对应 `@font-face`，没有假性加粗或回落。
- [ ] 数字+中文混排对齐正常。
- [ ] 最终 HTML 中没有 `fonts.googleapis.com`、`fonts.gstatic.com` 或其他远程字体链接。

## 6. 中文旁白 TTS

中文配音 / 旁白默认走豆包脚本，不用 `npx hyperframes tts` 的 Kokoro 中文音色。

```bash
python3 references/modules/hyperframes-media/scripts/doubao_tts.py \
  "你的中文旁白文案" --voice cancan -o narration.mp3
```

凭证来自运行环境变量（`DOUBAO_APP_ID` / `DOUBAO_ACCESS_TOKEN`）或显式外部配置路径，不存放在 skill 目录。缺凭证时停止并要求补运行环境配置，不要把密钥写进项目或 skill。

拿到 `narration.mp3` 后，照常用转写流程生成词级时间轴。中文转写优先用开沿的 `audio-transcribe` skill；若必须走 HyperFrames CLI，确认 Whisper 模型和语言参数不会把中文误翻译成英文。

## 7. 一句话

中文字体：本地 `.woff2` + `@font-face`，最终产物不留 Google Fonts。中文旁白：豆包脚本，凭证走环境变量，不进 skill。
