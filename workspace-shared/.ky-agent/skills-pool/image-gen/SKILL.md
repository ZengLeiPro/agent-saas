# Image Gen - AI 图像生成

> **品牌色规范**：主蓝 `#2E56E1`、暖橙 `#E8843A`、暖橙文字 `#A0500E`（不用 `#B65E16`）；中文字体 PingFang SC 系。完整品牌规范见 `brand-guidelines` skill。

所有图像生成一律**调用平台内置的 `GenerateImage` 工具**完成。本 skill 只提供方法论：交互流程、prompt 设计、风格库、尺寸选型与参考图用法。

## GenerateImage 工具速览

| 参数 | 说明 |
|------|------|
| `prompt` | 图像描述提示词（必需） |
| `model` | 引擎：`gpt-image-2`（默认，旗舰质量）或 `seedream`（备选，中文原生） |
| `aspectRatio` | `1:1`（默认）/ `16:9` / `9:16` / `4:3` / `3:4` / `3:2` / `2:3` |
| `quality` | `low` / `medium` / `high` / `auto`（默认，仅 gpt-image-2 生效） |
| `refImages` | 参考图（图生图）：workspace 相对路径数组，最多 4 张 |
| `count` | 生成张数，默认 1，最多 4 |

**计费**：每生成一张图按引擎单价扣组织积分（两个引擎单价不同，以平台当前定价为准）；组织积分余额不足时工具会直接拒绝并说明原因。生成失败不扣积分。平台不会自动切换引擎——切换引擎意味着不同扣费单价，必要时先告知用户再显式改 `model` 重试。

**产物**：工具把 PNG 写入工作区 `assets/generated/<日期>/` 下并返回相对路径。

### 核心交互流程

当用户提出图像生成需求时，按以下流程执行。目标：生成准确匹配用户意图的图片，而不是猜着画。

#### 第一步：确认 Prompt 模式

使用当前环境可用的用户提问工具向用户确认 prompt 方式；如果没有专用提问工具，则用一次简短提问完成确认。提供三个选项：

- **A. 帮忙设计提示语（继续提问）** — agent 给出风格/构图/色调等选项，用户挑选，agent 组装专业 prompt
- **B. 帮忙设计提示语（直接开始）** — agent 基于用户的简短描述自主扩展，不再追问，直接生成
- **C. 直接使用我的描述** — 用户的原文就是 prompt，原样发送给模型

在提问文本中加上选项建议，帮用户快速决策：
- 用户描述短/模糊（"画一只猫"、"做个封面"）→ 建议 A 或 B
- 用户描述已包含风格、构图、氛围等细节 → 建议 C

**快捷跳过**：如果用户明确说了"你来想"、"随便画"、"surprise me"之类的话，不需要提问，直接按 B（直接开始）处理。

#### 第二步：Prompt 准备

**选项 A — 继续提问：**

使用当前环境可用的用户提问方式**一次性**问完关键问题（合并为一次提问，不要逐个问）：
- 主体/场景：画面核心元素是什么？
- 风格偏好：写实摄影 / 插画 / 水彩 / 油画 / 3D渲染 / 扁平设计 / 动漫 / 像素风 等
- 色调/氛围：暖色/冷色/高对比/柔和/赛博朋克/复古 等
- 构图偏好：特写/中景/全景/俯视/仰视 等
- 用途（影响尺寸选择）：社交媒体 / PPT / 海报 / 壁纸 / 头像 等
- 在提问末尾注明："以上任何一项没有想法可以留空，由我来决定"

根据用户回答组装完整 prompt。

**选项 B — 直接开始：**

基于用户的简短描述，自主扩充为 50-150 字的完整画面描述。主动补充风格质感、光线色调、构图视角、氛围情绪等。

**选项 C — 原样使用：**

直接使用用户的原文作为 prompt，不做任何修改。

#### 第三步：生成

- **选项 A 和 B**：先在回复中展示你设计的完整 prompt，然后**立即调用 GenerateImage 工具生成**（展示是为了透明，不是等审批；用户如有异议会主动说，到时再调整重来）
- **选项 C**：直接调用 GenerateImage 工具生成，无需额外展示
- 根据用途自动选择合适的 `aspectRatio` 和 `quality`

#### 第四步：展示结果

1. 用 markdown 图片语法内联展示工具返回的相对路径：`![描述](assets/generated/yyyymmdd/img-xxxxxxxx.png)`
2. 如果工具返回了 `revisedPrompt`（模型重写后的 prompt），展示出来供参考
3. 一句话提示后续可选操作：调整描述重新生成 / 换风格或尺寸 / 生成多张变体

### Prompt 语言

- **gpt-image-2**：中英文均可，英文 prompt 通常更稳定（对英文构图词敏感）
- **seedream**：始终用中文（中文原生模型）

### 尺寸 × 用途对照

| 用途 | 推荐 aspectRatio |
|------|-----------------|
| 社交媒体配图 / 头像 | `1:1` |
| 横版封面 / Banner | `16:9` |
| 手机壁纸 / 竖版海报 | `9:16` |
| PPT 配图 | `4:3` |
| 摄影感画面 | `3:2`（横）/ `2:3`（竖） |

**质量智选**（gpt-image-2）：信息图/海报/插画 → `quality: "high"`，社交媒体配图 → `"low"`，不确定 → `"auto"`。

### 参考图（图生图）

当用户上传了图片时，对话上下文中会出现类似"用户上传了一张图片，路径是 uploads/example.jpg"的系统提示。agent 应：

1. 识别出这是参考图，调用 GenerateImage 时把该 workspace 相对路径放进 `refImages` 数组
2. 多张参考图就传多个元素：`refImages: ["uploads/a.jpg", "uploads/b.jpg"]`（最多 4 张）
3. 在 prompt 里直接描述要改什么（如"把图中的苹果改成绿色，其它保持不变"）

参考图会把图片内容发送到外部图像模型。图片包含客户资料、员工/个人照片、合同截图、未公开设计稿等敏感内容时，先说明将使用外部图像服务并取得用户明确授权，再调用工具。

### 其他规则

- **多图生成**：用户要多方案/变体时用 `count` 参数（最多 4），每张图都内联展示；注意每张都扣积分，先跟用户确认张数
- **错误处理**：工具报余额不足 → 如实告知用户并停止，不要换引擎硬试；工具报引擎暂不可用 → 说明情况，经用户同意后可显式改 `model: "seedream"` 重试（单价不同）
- **不重复确认**：第一步已做过意图确认，后续不要再问"确定吗""要生成吗"

## 风格 Prompt 参考库

`references/` 目录下存放了经过验证的风格 prompt 模板，可在设计 prompt 时参考或直接拼接使用：

| 文件 | 风格 | 适用场景 |
|------|------|---------|
| `references/gradient-glass.md` | 渐变毛玻璃卡片 | 科技产品、数据报告、Apple Keynote 风 |
| `references/ticket.md` | 数字极简票券 | 信息图表、黑白高级感、杂志排版 |
| `references/vector-illustration.md` | 扁平矢量插画 | 教育内容、故事叙述、复古温暖风 |

使用方式：读取对应风格文件内容，作为 prompt 前缀拼接用户的具体内容描述。
