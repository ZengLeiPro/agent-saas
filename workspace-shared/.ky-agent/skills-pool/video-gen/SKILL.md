# Video Gen - AI 视频生成

调用火山引擎方舟平台 Seedance 2.0 系列模型，文生视频 / 图生视频 / 多模态参考生视频，输出 mp4。

## 后端模型

| 模型 | Model ID | 分辨率 | 适用场景 |
|------|----------|---------|---------|
| **pro（默认）** | `doubao-seedance-2-0-260128` | 480p / 720p / **1080p** | 商业级输出、关键场景、需要 1080p、Demo / 路演 / 客户交付 |
| fast | `doubao-seedance-2-0-fast-260128` | 480p / 720p | 快速试稿、批量变体、成本敏感 |

**两者能力一致**：文生视频 / 图生视频-首帧 / 图生视频-首尾帧 / 多模态参考生视频 / 编辑视频 / 延长视频 / 音画同生。差别只在 fast 不能出 1080p、生成速度更快、价格更低。

固定参数：帧率 24 fps，时长 4-15 秒，输出 mp4。

## 使用方式

```bash
python3 {SKILL_DIR}/scripts/generate.py "<prompt>" -o <output_dir> [options]
```

脚本会同步阻塞，先 POST 创建任务，按 30 秒间隔轮询直到 `succeeded`，再立即下载 mp4（视频 URL 24h 内有效，必须立即落盘）。

如果是 pro 1080p 或参考图生成，实际等待可能超过 3 分钟。生成前用一句人话说明「我开始生成，可能要几分钟」，避免用户误以为卡住。

### CLI 参数

| 参数 | 说明 |
|------|------|
| `prompt` | 视频生成提示词（位置参数，必需） |
| `-o, --output-dir` | 输出目录（必需，遵守工作区规范 `assets/yyyymmdd/`） |
| `-r, --resolution` | 分辨率 `480p` / `720p` / `1080p`，默认 `1080p`。fast 模型若传 1080p 自动降到 720p 并 warn |
| `-a, --aspect-ratio` | 宽高比 `16:9` / `4:3` / `1:1` / `3:4` / `9:16` / `21:9` / `adaptive`，默认 `adaptive`（模型按提示词/参考素材自适应） |
| `-d, --duration` | 时长（秒），4-15 整数；`-1`（默认）由模型智能选择最合适长度 |
| `-n, --num` | 同提示词不同 seed 的并发变体数量，1-5，默认 1。**视频远比图片贵，n>1 前必须先和用户确认** |
| `-m, --model` | 完整 model id（默认 pro） |
| `--fast` | 等价于 `-m doubao-seedance-2-0-fast-260128` |
| `--no-audio` | 关闭音频（默认有声：模型自动生成对话/音效/BGM） |
| `--watermark` | 开启右下角 AI 水印（默认关闭） |
| `--seed` | 随机种子。`-1` 随机；n>1 时按 `+i` 偏移生成多个 |
| `--ref-image PATH_OR_URL` | 参考图（本地或 URL），可多次。prompt 中用「图1」「图2」按传入顺序指代 |
| `--ref-video PATH_OR_URL` | 参考视频（仅 2.0 系列支持），可多次。prompt 中用「视频1」「视频2」指代 |
| `--ref-audio PATH_OR_URL` | 参考音频（仅 2.0 系列支持），可多次。**不可单独传入**，必须配合至少一个 `--ref-image` 或 `--ref-video` |
| `--last-frame` | 同时下载视频尾帧 png，用于做连续视频（首帧→视频→尾帧→下一视频首帧） |
| `--priority` | 任务优先级 0-9，越大越优先（同 endpoint 内生效），默认 0 |
| `--max-wait` | 单任务最大等待秒数，超时返回 timeout 状态，默认 1800（30 分钟） |
| `--poll-interval` | 任务状态轮询间隔秒数，默认 30（火山官方推荐） |
| `--callback-url` | 可选 webhook，任务状态变化时火山主动 POST 推送 |

## 核心交互流程

收到用户的视频生成需求时，按以下步骤走。目标：让模型按用户意图生成想要的视频，不是猜着拍。

绝大多数员工对"prompt 工程""镜头语言""宽高比"这些术语没概念。**默认假设用户什么都不懂**，用大白话引导，主动替他们做技术决策，只在真正影响成片的地方让用户选。

### 第一步：确认 prompt 模式（用大白话问）

用 **AskUserQuestion 工具**问用户一个问题，三个选项：

- **A. 帮我想画面（你来问我几个问题）** — 你给出几个简单选项让用户挑，再组装成专业 prompt
- **B. 帮我想画面（直接开拍）** — 你基于用户说的一句话自由扩充，不再追问
- **C. 我已经想好画面了，照我说的拍** — 用户原文就是 prompt

判断建议（默认推荐 B，最快出片）：
- 用户描述只有几个字（「拍个山」「做个产品视频」）→ 推荐 B 直接开拍
- 用户已经写了场景、动作、氛围 → 推荐 C
- 用户明确要慢慢打磨细节 → 用 A
- 用户说「你来定」「随便」「看你的」→ 跳过提问，直接按 B
- 用户上传图片并说「以这张图」「让图动起来」「按这张图做」→ 跳过提问，直接按 B；先定位并确认参考图，再生成

### 第二步：Prompt 准备

**选项 A — 一次性问完用户能感知到的关键问题**（不要逐个问）。问题措辞必须是员工能秒懂的人话，不要出现"运镜""景别""色调"等术语：

- **画面里有什么？** 谁/什么东西，在哪里，在干什么？例子：「我们公司的新会议室，几个销售围在桌子边讨论方案，桌上有笔记本电脑」
- **想要什么感觉？**（选项给员工挑）：
  - 高级商务 / 温暖治愈 / 科技感冷感 / 复古怀旧 / 童趣可爱 / 大气电影感
- **画面节奏？**（选项给员工挑）：
  - 慢一点，氛围片，看着舒服
  - 中等节奏，正常叙事
  - 快切快剪，抓眼球
- **要不要声音？**（选项给员工挑）：
  - 要人物对话（如果要，让用户直接给台词原文）
  - 只要 BGM / 音效，不要人说话
  - 完全无声，我自己后期配
- **发哪里用？**（选项给员工挑）：
  - 朋友圈 / 抖音 / 视频号 / 小红书短视频 → 9:16 竖屏
  - 公众号 / B 站 / YouTube / 官网 / 给客户看 → 16:9 横屏
  - 大屏展示 / 电影感片头 → 21:9 超宽
  - 表情包 / 头像 → 1:1
- **有没有现成的图、视频、音乐可以参考？**（如果有，让用户提供路径或上传）
- 提问末尾必须注明：「以上任何一项没想法都可以留空，我来定」

**选项 B — 直接开拍**：基于用户的一句话，自主扩展为 80-200 字的完整 prompt。你自己定镜头、节奏、氛围、音频策略；按用户场景智选宽高比和时长。

如果用户上传了图片但没有给出明确文件路径，先用 `Glob` 查找 `uploads/*` 中最近的候选图片，再用 `Read` 打开确认画面内容，确保用的是本轮用户上传的图，而不是历史旧图。确认后用 `--ref-image` 传入，并在 prompt 中用「图1」指代。

**选项 C — 原样使用**：直接用用户原文。

### 第三步：生成

- **A / B**：先在回复中用大白话**复述一遍你的理解**（不要把"prompt"二字甩给用户看，员工看了头大），比如「我打算这样拍：xxx，xxx，xxx。3 分钟后给你视频」，然后**立即调用脚本生成**——不要等审批，用户如有异议会主动说。
- **C**：直接生成，不额外展示理解。
- **自己决定**所有技术参数（分辨率/宽高比/时长/seed/水印）。员工没说就用默认+智选。
- **n > 1 务必先确认**：视频比图片贵很多，要做 3 个变体之前必须再确认一次「想要 3 个不同版本对比吗？这会消耗 3 倍时间和费用」。
- **告知预计等待时间**：单视频一般 1-3 分钟（fast 模型）到 5-8 分钟（pro 1080p）。**提前告知**避免用户以为卡住了。
- **图生小动效默认无声**：用户只是想让实物图、产品图、玩偶图轻微动起来时，默认加 `--no-audio`。原因是模型自动配音/音效容易抢戏，且可能生成不符合画面的声音；用户明确要 BGM、环境音、音效或对白时再开音频。

### 第四步：展示结果

视频生成完成后，用 **markdown 视频内联语法**展示，系统会自动渲染为内联播放器：

```markdown
![描述这段视频是什么](assets/20260527/seedance_1716796800.mp4)
```

如果有尾帧（`--last-frame`）一起展示。**不要**输出 `[FILE]` 标记——视频已经能内联播放，再来个下载卡片是冗余。

最后给一行**人话提示**后续可选操作，挑 2-3 个最相关的：「不满意可以让我换个氛围/换个角度重做」「想要竖屏版/横屏版直接告诉我」「想要 3 个不同版本对比也可以」「这个视频的最后一帧能用作下一段视频的开头，做成连续剧情」。

## Seedance Prompt 写作要点

### 镜头语言
直接写进 prompt，模型对镜头术语理解力很强：
- 视角：**第一人称视角** / 第三人称 / 主观镜头 / 上帝视角
- 景别：**特写 / 近景 / 中景 / 全景 / 远景**
- 运镜：**推（dolly in）/ 拉（dolly out）/ 摇 / 移 / 跟随 / 环绕 / 升降**
- 角度：俯视 / 仰视 / 平视 / 倾斜（dutch angle）

### 时长卡点（强烈推荐）

对超过 5 秒的视频，**按秒分段描述**显著提高节奏控制力，火山官方示例都用了这种结构：

```
0-2 秒：你的手摘下一颗带晨露的红苹果，轻脆的苹果碰撞声；
2-4 秒：快速切镜，苹果块投入雪克杯，加入冰块与茶底，用力摇晃；
4-6 秒：第一人称成品特写，分层果茶倒入透明杯；
6-8 秒：手持举杯到镜头前，背景音"来一口鲜爽"。
```

### 音频提示（`generate_audio=true` 时）

- **对话**用**双引号包裹**：`男人叫住女人说："你记住，以后不可以用手指指月亮。"`
- **音效**直接描述：`木门吱呀声、远处犬吠`
- **BGM** 用自然语言：`轻快电子鼓点贯穿全程` / `钢琴独奏，缓慢忧伤`
- 生成的有声视频均为单声道。

### 参考素材的 prompt 指代

`--ref-image` / `--ref-video` / `--ref-audio` 的**传入顺序 = 编号顺序**。prompt 中**必须用「图1」「图2」「视频1」「音频1」**来指代：

```
首帧使用图1，尾帧使用图2，参考视频1的运镜风格，BGM 用音频1。
```

## 参考素材规则

### ⚠️ 真人人脸限制（必须告知用户）

Seedance 2.0 系列**不支持直接上传含真人人脸的参考图/视频**。如果用户传入真人照片做参考，应在 prompt 阶段提醒用户，可选：

1. 使用平台预置的虚拟人像（火山官方提供素材）
2. 使用本账号近 30 天内由 Seedance 2.0 生成的人脸视频/尾帧作为新一轮参考
3. 使用平台已授权的真人素材
4. 让 AI 用文字描述人物外形（年龄/性别/服饰/发型），由模型自由生成

脚本不强制拦截，但提交后 API 可能返回审核拒绝。**在调脚本前主动告知，避免无意义消耗**。

### 本地素材 vs URL

- **本地路径**：脚本自动转 base64 data URI。建议单文件 ≤ 5MB（base64 编码后体积会增大 33%，请求体过大会失败）
- **URL**：必须**公网可访问**（火山服务器要能 GET 到），不接受 localhost / 内网 / 需登录的私链
- 支持的格式：
  - 图片：jpg / jpeg / png / gif / webp / bmp
  - 视频：mp4 / mov / webm / m4v / mkv
  - 音频：mp3 / wav / m4a / aac / ogg / flac

### 组合规则

- 文本 prompt 可以单独传
- 图 / 视频 / 音频 可任意组合作为参考
- **音频不能单独传**，必须配合至少 1 个图或视频

## 错误处理

脚本在 HTTP 4xx/5xx 时把 API 返回的 `error.code` / `error.message` 原样打印到 stderr。

常见错误：
- `401 Unauthorized` → ARK_API_KEY 未设置或失效
- `403 Forbidden` → 账户余额不足或模型未开通
- `404 ModelNotOpen` → 当前方舟账号未开通指定 Seedance 模型服务。先判断失败的是哪个模型：如果 pro `doubao-seedance-2-0-260128` 未开通，可自动改用 `--fast -r 720p` 试一次；如果 fast `doubao-seedance-2-0-fast-260128` 也未开通，再告知用户需要在方舟控制台开通模型。开通后可原命令重试，不需要改 prompt。
- `400 InvalidParameter` → 参数取值不合法（resolution 与 model 不匹配等）
- `429 TooManyRequests` → 触发限流，降低 `-n` 或加 `--priority`
- 任务 status=`failed` → 通常是 prompt 触发审核（真人人脸 / 违规内容）
- 任务 status=`expired` → 超过 `execution_expires_after`（默认 48h），通常不会发生

部分任务失败时（多任务模式），已成功的视频仍然保留，进程以 exit code 2 退出，告知用户「3 个里成功 2 个」。
