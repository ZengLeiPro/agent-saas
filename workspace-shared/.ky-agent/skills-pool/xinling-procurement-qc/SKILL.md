# 新凌公司采购+品管数据查询

数据来自新凌公司 2026 年 1-4 月份的品管部数据（含部分历史累积）。所有原始 Excel 已预处理为单个 SQLite 数据库 `data/procurement_qc.db`，包含 12 张业务表。Agent 通过 SQL 查询回答用户问题。

## 工作流（每次会话首次使用时）

```bash
# 1. 看表结构（必读）
SKILL_DIR="<path-to-this-skill-directory>"
cat "$SKILL_DIR/schema.md"

# 2. 查询数据
sqlite3 -header -column "$SKILL_DIR/data/procurement_qc.db" "<SQL>"
```

或用 Python：

```python
import sqlite3
from pathlib import Path

skill_dir = Path("<path-to-this-skill-directory>")
conn = sqlite3.connect(skill_dir / "data" / "procurement_qc.db")
df = pd.read_sql_query("...", conn)
```

注意：使用当前加载到的 skill 目录作为 `SKILL_DIR`。不要硬编码 `.claude/skills/...` 或 `skills-pool/...`；如果运行环境无法直接给出 skill 目录，先用 `rg --files`/目录列表定位包含本 `SKILL.md`、`schema.md` 和 `data/procurement_qc.db` 的目录。

## 核心原则

### 1. 先读 schema 再写 SQL
绝不凭表名猜字段。每次会话第一次用本 skill 时，**必须先 `cat schema.md`** 把所有表名+字段+业务术语装入上下文。

### 2. 分析像懂行的助理，而不是数据库执行器
用户问"3 月皮胚检验最差的是什么"时，不要只 SELECT 一条记录返回——主动：
- 算汇总（不良率/呆滞占比的均值、Top10）
- 拉趋势（与 1/2 月对比，是否有恶化）
- 找异常（是否集中某个客户/品种/皮源）
- 给结论（"3 月皮胚整体呆滞占比 X%，主要拖累是 Y 客户的 Z 品种，与 2 月相比上升 N pp"）

### 3. 业务术语必须用对（详见 schema.md "业务术语"段）
- **皮胚** = 半成品（脱毛+鞣制后未涂饰）
- **成品** = 涂饰完工的皮料
- **皮源** = 原料/半成品的来源批次（如 `MP260330JBS3150` 是批次编号）
- **二层** = 片皮后的底层皮料（次质，单独计费）
- **呆滞** = 长期不动用的库存或检验中识别的低品质留库部分
- **片皮** = 把厚生皮按厚度分层切开

### 4. 数据汇报零幻觉（沿袭主硬规则）
任何带客户名/合同号/金额/日期/数量/姓名的具体表述都必须从数据库现查现得，不允许"印象推断"。如果某个表的具体行需要复核，用 `raw_row` 字段对照原始 Excel：

```bash
# raw_row 是原始 sheet 的行号（1-based），可回查 source_xlsx/ 下对应文件
```

### 5. 数据时间口径
- **当年（2026 年 1-4 月）**：4 份品检日报里 `品检_皮胚检验` / `品检_成品检验` 主要内容、`采购_皮源总表` 含部分历史合同
- **跨年历史累积**：`品检_皮源入库累积`（2022.07-至今）、`品检_库存月度快照`（2025.08/09/10 三月并排）
- 注意：`品检_皮胚检验` 里部分行的日期是 2025 年（人工挑选放进当月日报的历史问题件）

## 典型查询入口

详见 `examples.md`（5-10 个真实用例）。常见问题套路：

| 用户问题类型 | 优先查的表 |
|--------------|---------|
| "X 客户 X 月份的品检情况" | 品检_皮胚检验 / 品检_成品检验 |

| "呆滞消化（出库/套染）" | 品检_呆滞消化 |
| "皮胚库存水位" | 品检_库存月度快照（仅 8/9/10 月） |

## 数据更新

数据为一次性快照（2026-05-14 构建）。如需更新：
1. 替换 `source_xlsx/` 下原始 Excel
2. 运行 `python3 scripts/build_db.py`
3. 注意 sheet 名变化可能需要修改解析逻辑（参考 schema.md 末尾的"已知噪声"段）
