# 新凌采购+品管数据集 Schema

> 一次性快照（构建于 2026-05-14）。所有日期已尽量规整为 `YYYY-MM-DD`；少数无法解析的保留原字符串。
> 大部分中文表名/字段名直接使用——SQL 查询时给中文标识符加双引号或反引号。

## 业务术语（必读）

| 术语 | 含义 |
|------|------|
| 皮源 | 原料/半成品的来源批次。两种形态：①原始毛皮的"代理商-合同号-产地"（如 A1178 法国阉牛）；②已加工的批次编号（如 `MP260330JBS3150`、`LP260107LY507`）。**这是贯穿采购→片皮→检验全链路的关键标识符** |
| 毛皮 | 未脱毛的原始皮，进口居多（法国阉牛/西班牙阉牛/塔斯马尼亚等） |
| 蓝皮 / 皮胚 | 鞣制后未涂饰的半成品（蓝色） |
| 成品 | 涂饰完工的最终皮料 |
| 片皮 | 把厚生皮按厚度（如 2.6-2.8mm）水平切开 |
| 二层 | 片皮后产生的底层皮料（次质，单独计费、独立成本） |
| 灰皮 | 脱毛、未鞣制的中间形态 |
| 呆滞 | 长期不动的库存，或检验中识别需留库观察的批次/片数 |
| 呆滞占比 | 单次检验中呆滞数量 ÷ 检验张数 |
| 套染 | 把呆滞库存再次染色复用，呆滞消化的一种方式 |
| 鼓 / 进鼓 | 转鼓加工工序，"进鼓条数"指进入下一工序的有效条数 |
| 张/条/片 | 计量单位。生皮按"条"或"张"，半成品/成品按"片"，重量按 KG/T，面积按 SF（平方英尺） |

## 客户编号

`GD001/GD209/GD210/GD232`（家具/沙发类）、`WZ020/WZ008/WZ147`（鞋面/服装类）、`BD001`、其他——具体业务归属以实际数据为准。

## 表结构详解

### 采购_皮源总表（54 行，明细）

2026 年皮源入库主明细。每行 = 一个皮源批次。

| 字段 | 类型 | 含义 |
|------|------|------|
| in_date | TEXT | 入库日期 YYYY-MM-DD（forward-filled，合并单元格首行的日期） |
| supplier | TEXT | 供应商（如刘洋/兴龙皮革/何丹枫/信亿） |
| contract_no | TEXT | 合同号/采购方向描述（如 `外购国产公牛+中小蓝皮`、`JX03/外购意大利蓝皮`） |
| pi_yuan | TEXT | 皮源（最关键字段，含批次编号，如 `外购美国轻磅奶牛蓝皮/LP260107LY507`） |
| net_weight_t | REAL | 净重（吨） |
| container_no | TEXT | 车号/柜号 |
| rolls | REAL | 条数 |
| boards | REAL | 板数 |
| stock_rolls | TEXT | 库存条数（可能为字符串） |
| drum_rolls | TEXT | 进鼓条数 |
| loss_rolls | TEXT | 报损数 |
| blue_rolls | REAL | 蓝皮成品条数 |
| blue_sf | REAL | 蓝皮成品 SF（平方英尺） |
| raw_row | INTEGER | 原 Excel 行号（1-based，便于回查） |

### 采购_皮源汇总（54 行，小计+合计）

非明细行（pi_yuan 为空但有数量值）。`row_type` 字段尝试区分：
- `合同小计` — 合同内的子合计或尺码分组合计
- `年度合计` — 年度滚动总数（按 rolls ≥ 1000 简单启发，可能不完全准确）

⚠️ row_type 判定逻辑较粗，需要严格区分时以 `raw_row` 回查 `源 xlsx/皮源.xlsx` 原始 sheet 为准。

### 进口毛皮_合同安排（5 行）

进口毛皮采购的合同级总安排。每行 = 一个合同的一个规格。

| 字段 | 含义 |
|------|------|
| agent | 代理商（MAIVIS LIMITED/迈维斯、何丹枫、柯卓、宛学义、许鹏程） |
| region | 代理商所在地区 |
| factory | 毛皮生产厂家 |
| factory_region | 厂家所在地区 |
| contract_no | 合同号（如 A1178 / A2738） |
| fur_source | 皮源类型（法国阉牛/西班牙阉牛/波兰阉牛等） |
| spec | 规格（如 15KG、9-10KG、25-30KG 平均 26KG） |
| total_weight | 总重量 KG |
| rolls | 条数 |
| arrival_date | 到场时间 YYYY-MM-DD |
| processing_site | 加工点（富洋/俊安） |
| benchmark_product | 对标产品 |
| split_thickness | 片皮厚度 |

### 进口毛皮_批次加工（10 行）

每行 = 一批毛皮的片皮加工记录（含合同号、片皮工艺、二层产出与费用）。

| 字段 | 含义 |
|------|------|
| sheet_name | 原始 sheet 名（即批次的简称，如 `何丹枫南非阉牛1800`） |
| 加工场地 | 富洋/俊安 |
| 片皮时间 | 片皮日期 |
| 负责人 | 加工负责人（多为"王亮"） |
| 到厂日期 | 毛皮到加工场地的日期 |
| 合同号 | 关联进口合同号（可与 `进口毛皮_合同安排` JOIN） |
| 批次编号 | 内部批次号（如 `MP260330JBS3150`） |
| 生产日期 | 实际生产开始日期 |
| 毛皮代理商 | 同 `合同安排.agent` |
| 毛皮名称 | 同 `合同安排.fur_source` 但措辞略异（如"BYD3#"、"TA1(塔斯马尼亚毛皮)"） |
| 片皮厚度_1批 | 第一批片皮厚度（如 `2.6-2.8`） |
| 片皮厚度_2批 | 第二批厚度（如有） |
| 选出条数_灰皮 | 分灰皮后选出的有效条数 |
| 送货张数_或实际张数 | 最终送货/实际张数（不同卡片对应字段名不同：送货张数/实际张数/总张数/到柜张数） |
| 二层第一批重量 | KG |
| 二层第二批重量 | KG（部分批次空） |
| 二层总费用 | 元 |
| 二层平均KG_per_条 | 每条二层平均重量 |
| 二层平均元_per_条 | 每条二层平均费用 |

### 品检_皮胚检验（410 行，核心明细表）⭐ 已清洗

4 个月份皮胚检验记录合并。**每订单 1 行**（已合并源 Excel 中的"皮胚汇总主表"+"皮胚汇总 (3)/2) 拆分表"，无 source_sheet 区分；直接 SUM 即可得正确呆滞总数，不会双倍）。

| 字段 | 含义 |
|------|------|
| month | 月份 1/2/3/4（来自源文件名） |
| 类别 | 包袋 / 鞋面 / 沙发 / 服装 等 |
| 日期 | 检验日期 |
| 客户 | 客户编号 |
| 单号 | 订单单号 |
| 品种 | 黄牛 P5 / 黄牛 W-9（手掌纹）/ 黄牛磨砂等 |
| 颜色 | 文本（含色号） |
| 皮源 | 原始皮源文本（自由格式，保留） |
| **皮源_批次编号** | 从 `皮源` 提取的批次（如 `MP250625HYG1800`）。**89.7% 行覆盖** |
| **皮源_代理商** | 从批次中提取的字母段（HYG/LY/WSX/N/YGN 等）。直接 `WHERE 皮源_代理商='HYG'` 即可 |
| **皮源_产地** | 英国/法国/西班牙/美国/国产 等关键词识别 |
| **皮源_原皮类型** | 奶阉/阉牛/公牛/母牛/小牛/未育母牛 等 |
| **皮源_等级** | E级/8级/1#/ABC 等 |
| 厚度 | 如 `1.35-1.4` |
| 检验张数 | 检验数量（部分含 0.5） |
| 尺码SF | 尺码总和（SF 平方英尺） |
| 呆滞占比 | 呆滞 ÷ 检验张数（0-1） |
| 异常数量 | （主表中长期为空，业务上未启用） |
| 呆滞数量 | 实际留库观察的呆滞数 |
| **缺陷分类** | 从拆分表抽取的缺陷+数量明细（如 `松面:19;面差:9`）。当订单有呆滞且备注规范时填充，128/410 行覆盖 |
| 备注 | 人工填写说明（如"挖刀8条留下"、"皮胚面差46条留下"） |
| raw_row | 原 sheet 行号 |

### 品检_成品检验（623 行）

字段同皮胚检验，但 `不良率` 替代 `呆滞占比`，无 `缺陷分类`。皮源字段同样已拆为 5 个结构化字段。

### 品检_皮胚发胚日报（18 行）+ 品检_成品发胚日报（61 行）

发胚/发货日报。皮胚日报字段：序号/订单号/客户/品种/颜色/厚度/片数/尺码/皮源/品检员/备注；成品日报字段：序号/客户/品种/颜色/数量/技术员/皮源/备注。

⚠️ 4 月份源 .xls 没有这两个 sheet（只 1、3 月有，且 1 月==2 月）。

### 品检_呆滞特例（12 行）

`皮胚汇总 (2)` 提取的人工挑选呆滞案例，比标准检验表多 `订单负责人 / 品检员 / 工序 / 责任人` 字段。多个客户/订单的复合复盘。

### 品检_呆滞消化（140 行）⭐ 已清洗

呆滞库存的消化方式跟踪。`消化类型` = `出库` 或 `套染`。

**已修复源 Excel 三月列结构不一致问题**（1 月 13 列、3 月 19 列、4 月 15 列），脚本按月份分别配置列号映射，输出统一 schema：

| 字段 | 含义 |
|------|------|
| month | 1 / 3 / 4（2 月与 1 月数据一致，跳过） |
| 消化类型 | 出库 / 套染 |
| 日期 | 消化日期 |
| 类别 | 包袋 / 沙发 等（1 月源 sheet 没有此字段，为 NULL） |
| 客户 | 客户编号（仅 3 月有；1、4 月为 NULL） |
| 品种 | 品种名 |
| 颜色 | 颜色 |
| 数量_条 | 条数 |
| 数量_SF | 平方英尺 |
| 皮源 | 自由文本，已同步拆为 5 个 `皮源_xxx` 字段 |
| 原责任 | 仅 3 月有；1、4 月为 NULL |
| raw_row | 原 sheet 行号 |

### 品检_皮源入库累积（39 行）

历史皮源入库累积（2022-07 至今）。字段：日期/名称/级别/张数/尺码SF。

注：与采购侧的 `采购_皮源总表` 是**两套独立来源**——本表是品管部按级别记录的入库流水，采购侧是按合同入库流水，可能有重叠但口径不同。

### 品检_库存月度快照（66 行）

2025 年 8/9/10 月的皮胚库存分析。字段：month/品类/单位/数量/SF/变动数量/变动SF。品类 = 黄牛成品/黄牛外购/水牛皮胚/水牛成品/犀牛皮胚/犀牛成品/犀牛外购/新增备胚/新增呆滞/总计/总库存/黄牛历史库存。单位 = 条 或 片。

## 已知噪声与坑

1. **皮胚检验有 2025 年日期**：少数行的"日期"是 2025 年（人工挑选的历史问题件复盘）。如需严格"本月新增"，按 `month` 字段而非 `日期` 字段筛选。
2. **采购_皮源汇总 row_type 不准**：判定逻辑简化版，严格区分以 `raw_row` 回查原 Excel 为准。
3. **进口毛皮_批次加工字段不齐**：每张卡片 sheet 命名约定不一致（送货张数/实际张数/总张数/到柜张数 等），脚本择优合并到 `送货张数_或实际张数`。
4. **客户编号未做实体化**：`GD001` `GD232` 等以字符串存储，无单独维表。
5. ~~**皮源字段是自由文本**~~ → **已拆分**：所有相关表都新增 `皮源_批次编号 / 皮源_代理商 / 皮源_产地 / 皮源_原皮类型 / 皮源_等级` 5 个结构化字段，原 `皮源` 字段保留作冗余。直接用这些字段精确匹配（如 `WHERE 皮源_代理商='HYG'`），不再需要 `LIKE '%xxx%'`。
6. **被弃用的 sheet**：`日报`（65519 假行模板）、`Sheet3`（空）、`日报 (2)`（跨 4 月一致的静态问题分类卡片）。
7. **2 月数据极稀疏**：仅 8 单 729 张（春节假期影响），跨月对比时 2 月会显得特别"安静"。

## 构建期 Sanity Check

`build_db.py` 末尾自动跑 4 条硬约束，构建失败会输出告警：

1. 呆滞消化 `数量_条 > 数量_SF` 且 > 1000 行（防止字段错位回潮）
2. 皮胚检验 `呆滞数量 > 检验张数`（业务上不可能）
3. 皮胚检验 `呆滞数量 / 检验张数 ≈ 呆滞占比`（差 >5pp 告警）
4. 整体加权呆滞率应在 5-15% 合理区间内（当前 8.11%）
